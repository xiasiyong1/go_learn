# 024. 泛型

## 问题

Go 泛型解决什么问题？什么时候不应该用泛型？

## 核心答案

泛型用于表达“相同逻辑适用于多个类型”的场景，减少重复代码，同时保留类型安全。典型场景包括容器、算法、通用工具函数。

不应该为了少写几行代码强行使用泛型。如果接口、普通函数或明确的业务类型更清楚，就不需要泛型。

## 代码示例

```go
func Max[T cmp.Ordered](a, b T) T {
	if a > b {
		return a
	}
	return b
}
```
