# 062. 基础类型

## 问题

Go 的基础数值类型、类型转换和溢出规则有哪些需要注意？

## 先给结论

Go 不会在不同数值类型之间做隐式转换，类型转换必须显式写出。整数溢出、无类型常量、浮点精度和 rune/byte 的底层类型，是初学者最容易混淆的几个点。

## 深入理解

### 1. 这道题真正考察什么

- 是否知道 `int`、`int64`、`uint` 是不同类型，不能直接赋值。
- 是否能解释无类型常量为什么更灵活。
- 是否知道整数转换可能截断或回绕。
- 是否能区分 `byte`、`rune` 和普通整数别名。

### 2. 底层机制要讲清楚

- Go 的赋值规则强调类型明确，减少隐式转换带来的意外精度或符号问题。
- 无类型常量在赋值或传参时才根据上下文确定具体类型。
- 整数类型转换按目标类型宽度取值，超出范围不会自动报运行时错误。
- 浮点数是近似表示，不适合直接表达金额等需要十进制定点精度的数据。

### 3. 工程实践怎么取舍

- 跨边界数据使用明确宽度类型，例如协议字段用 `int32`、`int64`。
- 索引和长度通常使用 `int`，避免无谓转换。
- 金额使用整数最小单位或 decimal 库，不使用 float64 做精确账务。
- 类型转换前检查范围，尤其是从大整数转小整数或有符号转无符号。

### 4. 常见误区

- `uint(-1)` 这类转换会得到很大的无符号数。
- 把 `len(s)` 的 int 随意转成 int32，数据大时可能截断。
- 比较浮点数时用 `==` 判断计算结果。
- 以为 `rune` 表示一个用户看到的字符，实际它只是 Unicode 码点。

## 如何验证理解

- 写边界值测试覆盖最大值、最小值和越界转换。
- 用 `fmt.Printf("%T %v")` 观察常量和变量的具体类型。
- 对金额、时间戳、ID 等关键字段增加解析和范围测试。
