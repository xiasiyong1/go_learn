# 019. for range 常见坑

## 问题

`for range` 有哪些常见坑？

## 核心答案

`for range` 遍历时拿到的 value 是元素副本，不是原元素本身。修改 value 不会修改切片或数组中的元素。

对 map 遍历时顺序不保证稳定。

闭包中捕获循环变量曾经是常见坑。Go 1.22 起，`for` 循环变量语义已调整，常见闭包捕获问题减少，但阅读旧代码和兼容旧版本时仍要注意。

## 代码示例

```go
users := []User{{Name: "a"}}
for _, u := range users {
	u.Name = "b" // 不会修改 users[0]
}
```

- 遍历 map 时删除元素是否安全？
