# 052. 减少内存分配

## 问题

如何减少 Go 程序的内存分配？

## 先给结论

减少分配的核心不是技巧堆叠，而是找到热点路径中的不必要对象、逃逸、扩容和转换。优化要以 benchmark 和 pprof 为依据。

## 深入理解

### 1. 这道题真正考察什么

- 是否能用 `-benchmem` 找到分配次数和字节数。
- 是否知道预分配 slice/map 能减少扩容。
- 是否能识别 string/[]byte 转换、接口装箱、闭包捕获等隐性分配。
- 是否能权衡可读性和性能。

### 2. 底层机制要讲清楚

- 每次堆分配都会增加分配器工作和后续 GC 压力。
- slice/map 扩容会分配新存储并搬迁数据。
- 接口装箱、返回指针、闭包捕获可能让对象逃逸到堆。
- 字符串拼接反复创建新字符串，Builder 或 Buffer 可以减少临时对象。

### 3. 工程实践怎么取舍

- 先定位热点，再优化最大分配来源。
- 使用 `make` 预估容量，减少扩容。
- 复用 buffer 时清理状态，并避免跨 goroutine 共享风险。
- 对字符串构造用 `strings.Builder`，对字节协议用 `bytes.Buffer` 或预分配 []byte。

### 4. 常见误区

- 为了减少一个分配写出复杂难懂的代码。
- 只看分配次数，不看总耗时和尾延迟。
- 复用对象时忘记清理，产生数据污染。
- 用 unsafe 零拷贝破坏不可变性或对象生命周期。

## 如何验证理解

- 用 `go test -bench . -benchmem` 对比改动。
- 用 `pprof -alloc_space` 找累计分配热点。
- 用 `-gcflags=-m` 验证逃逸变化。

