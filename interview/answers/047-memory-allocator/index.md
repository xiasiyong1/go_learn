# 047. 内存分配器

## 问题

Go 内存分配器大致怎么工作？小对象和大对象有什么区别？

## 核心答案

Go 内存分配器按大小规格管理对象。小对象通常从 per-P 的缓存中快速分配，减少全局锁竞争；大对象会走更重的分配路径。

对象越多、指针越多、生命周期越长，GC 扫描和管理成本通常越高。

## 代码示例

```go
buf := make([]byte, 0, 4096) // 预分配可减少重复扩容
```
