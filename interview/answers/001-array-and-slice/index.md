# 001. 数组和切片

## 问题

Go 中数组和切片有什么区别？切片扩容时需要注意什么？

## 先给结论

数组是定长值类型，长度是类型的一部分；切片是对底层数组的一段动态视图，包含指针、长度和容量。切片传参和赋值只复制切片头，不复制底层数组。`append` 可能复用原数组，也可能分配新数组，所以扩容和共享底层数组是这题最核心的追问点。

## 1. 数组长度属于类型的一部分

`[3]int` 和 `[4]int` 是不同类型。

```go
var a [3]int
var b [4]int

// a = b // 编译失败：类型不同

fmt.Println(a, b)
```

数组赋值会复制整个数组。

```go
a := [3]int{1, 2, 3}
b := a
b[0] = 100

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [100 2 3]
```

函数传数组也是复制。

```go
func modify(a [3]int) {
	a[0] = 100
}

func main() {
	a := [3]int{1, 2, 3}
	modify(a)
	fmt.Println(a) // [1 2 3]
}
```

如果要修改原数组，需要传指针。

```go
func modify(a *[3]int) {
	a[0] = 100
}
```

## 2. 切片是底层数组的视图

切片可以理解为三部分：

```text
ptr -> 指向底层数组
len -> 当前可访问元素数量
cap -> 从 ptr 开始到底层数组末尾的容量
```

示例：

```go
a := [5]int{1, 2, 3, 4, 5}
s := a[1:3]

fmt.Println(s)      // [2 3]
fmt.Println(len(s)) // 2
fmt.Println(cap(s)) // 4，从 a[1] 到 a[4]
```

修改切片会影响底层数组。

```go
s[0] = 100
fmt.Println(a) // [1 100 3 4 5]
```

## 3. 切片赋值只复制切片头

```go
s1 := []int{1, 2, 3}
s2 := s1

s2[0] = 100

fmt.Println(s1) // [100 2 3]
fmt.Println(s2) // [100 2 3]
```

`s1` 和 `s2` 是两个切片头，但指向同一个底层数组。

这也是为什么切片传参能修改元素：

```go
func modify(s []int) {
	s[0] = 100
}

func main() {
	s := []int{1, 2, 3}
	modify(s)
	fmt.Println(s) // [100 2 3]
}
```

但如果函数内部 append 后发生扩容，外部切片不一定看到新数组。

## 4. append 容量够时会复用底层数组

```go
a := []int{1, 2, 3}
b := a[:2] // len=2 cap=3

b = append(b, 100)

fmt.Println(a) // [1 2 100]
fmt.Println(b) // [1 2 100]
```

`b` 的容量够，所以 `append` 直接写进原底层数组的第 3 个位置，覆盖了 `a[2]`。

## 5. append 容量不够时会分配新数组

```go
a := []int{1, 2, 3}
b := a[:2:2] // len=2 cap=2，限制容量

b = append(b, 100)

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [1 2 100]
```

`a[:2:2]` 是完整切片表达式，第三个索引用来限制容量。因为 `b` 容量不够，append 会分配新数组，不会覆盖 `a`。

## 6. 保留小切片可能导致大数组不能释放

```go
func bad() []byte {
	data := make([]byte, 10<<20) // 10 MiB
	return data[:10]
}
```

返回的 10 字节切片仍然指向 10 MiB 底层数组，GC 会认为整个数组可达。

复制需要的部分：

```go
func good() []byte {
	data := make([]byte, 10<<20)
	out := append([]byte(nil), data[:10]...)
	return out
}
```

## 7. 面试时怎么答

可以这样回答：

- 数组是定长值类型，长度属于类型；赋值和传参会复制整个数组。
- 切片是底层数组的视图，切片头包含指针、长度、容量。
- 切片赋值和传参复制的是切片头，底层数组仍共享。
- `append` 容量够时原地写，容量不够时分配新数组。
- 多个切片共享底层数组时，一个切片 append 或修改可能影响另一个。
- 可以用完整切片表达式限制容量，或用 copy/append 复制来解除共享。
- 子切片长期持有大数组时，可能造成内存滞留。
