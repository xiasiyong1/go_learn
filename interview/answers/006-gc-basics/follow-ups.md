# 006. Go GC 基础 - 面试追问

## 追问与参考答案

### 1. 什么是逃逸分析？

逃逸分析是编译器判断变量是否能安全放在栈上的过程。如果变量可能在函数返回后继续被引用，或者编译器无法证明它只在当前函数内使用，就可能分配到堆上。可以用 `go build -gcflags="-m"` 查看结果。

### 2. `GOGC` 参数影响什么？

`GOGC` 控制下一次 GC 触发时堆增长比例，默认 100 表示新分配内存大约达到上次存活堆大小的 100% 后触发。调大通常减少 GC 频率但增加内存占用，调小则相反。

### 3. 为什么“减少分配”通常比“手动触发 GC”更重要？

减少分配会从源头减少垃圾数量、扫描成本和内存带宽压力。手动触发 GC 只能提前回收已经产生的垃圾，常常会增加停顿和 CPU 成本，不能替代降低分配。
