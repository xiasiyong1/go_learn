# 046. interface 底层实现

## 问题

interface 底层表示是什么？空接口和非空接口有什么区别？

## 先给结论

interface 底层保存动态类型信息和动态值。空接口和非空接口表示略有不同，但核心都是通过运行时类型信息支持动态派发、断言和 nil 判断。

## 深入理解

### 1. 这道题真正考察什么

- 是否能解释接口值为什么包含类型和值两部分。
- 是否知道空接口和非空接口在运行时表示上有差异。
- 是否能说明接口调用可能产生动态派发成本。
- 是否能解释接口装箱为什么可能导致逃逸。

### 2. 底层机制要讲清楚

- 空接口常被描述为 `(type, data)`。
- 非空接口需要记录具体类型如何满足接口方法集的信息，通常涉及 itab。
- 接口调用方法时需要通过动态类型找到对应方法实现。
- 把具体值放入接口可能需要复制或让值逃逸，以保证接口值生命周期正确。

### 3. 工程实践怎么取舍

- 小接口作为依赖边界有利于测试和替换。
- 热路径中避免不必要接口装箱，尤其是大值或频繁调用。
- 返回接口时注意 typed nil 问题。
- 接口设计应由消费者定义，表达需要的行为而不是生产者能力全集。

### 4. 常见误区

- 以为 interface 只是 void 指针，不理解方法集和动态类型。
- 把 nil 指针装入接口后判断接口不为 nil。
- 在性能热点大量使用 `interface{}` 参数导致分配和断言。
- 接口过大，导致实现类型被迫依赖无关方法。

## 如何验证理解

- 用 `%T` 打印接口动态类型。
- 用 `-gcflags=-m` 和 benchmark 观察接口装箱是否分配。
- 用编译期断言验证类型是否实现接口。

## 代码示例

```go
var p *MyError = nil
var err error = p
fmt.Println(err == nil) // false
```
