# 008. 零值

## 问题

Go 常见类型的零值是什么？零值可用有什么意义？

## 先给结论

零值是 Go 设计里非常重要的可用性原则：很多类型在未显式初始化时也能安全使用。但“零值可用”不是所有操作都可用，nil map、nil channel 等有明确边界。

## 深入理解

### 1. 这道题真正考察什么

- 是否知道不同类型的零值：数值为 0，布尔为 false，字符串为空串，指针、slice、map、chan、func、interface 为 nil。
- 是否能区分 nil slice 可 append、nil map 可读不可写、nil channel 收发永久阻塞。
- 是否知道 `sync.Mutex`、`bytes.Buffer` 等类型零值可直接使用。
- 是否能利用零值减少不必要构造函数和初始化分支。

### 2. 底层机制要讲清楚

- 零值由语言保证，不是运行时随机状态。
- slice 的 nil 零值没有底层数组，但 append 会按需分配。
- map 的 nil 零值没有哈希表结构，读返回元素零值，写入会 panic。
- channel 的 nil 零值没有通信对象，收发都会永久阻塞；在 select 中可用于动态禁用 case。

### 3. 工程实践怎么取舍

- 设计结构体时，让零值尽量处于合法、可关闭或可查询状态。
- 需要必填配置或外部资源时，仍然使用构造函数表达约束。
- 对集合返回值，按 API 语义决定 nil 和 empty，而不是机械追求零值。
- 在并发类型中尊重零值可用，不要要求调用方必须调用 Init。

### 4. 常见误区

- 把 nil map 当成空 map 写入。
- 把 nil channel 当成未初始化错误，却在 select 中误造成永久阻塞。
- 结构体零值含有 nil 指针字段，方法中直接解引用。
- 为了初始化所有字段写大量样板代码，反而破坏零值语义。

## 如何验证理解

- 为类型增加零值使用测试：声明变量后直接调用核心方法。
- 用表格测试覆盖 nil slice、empty slice、nil map 的边界。
- 代码 review 时检查构造函数是否真的必要。

## 代码示例

```go
var mu sync.Mutex
mu.Lock()
mu.Unlock()

var buf bytes.Buffer
buf.WriteString("hello")
```
