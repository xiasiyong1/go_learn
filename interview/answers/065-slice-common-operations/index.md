# 065. 切片操作

## 问题

Go 中切片的追加、删除、插入和过滤应该怎么写？

## 先给结论

切片没有内置的删除、插入 API，常见操作都由截取、`append`、`copy` 和重新切片组合出来。回答这题不能只背写法，还要说明：是否保持顺序、是否复用底层数组、是否会产生额外分配、是否需要清理尾部引用。

## 1. 追加：`append` 可能原地写，也可能换数组

最常见追加写法：

```go
s := []int{1, 2}
s = append(s, 3, 4)
fmt.Println(s) // [1 2 3 4]
```

如果容量足够，`append` 会复用原底层数组；容量不够才会分配新数组。

```go
a := make([]int, 2, 4)
a[0], a[1] = 1, 2

b := a
a = append(a, 3)

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [1 2]，但 b 和 a 仍共享底层数组
```

如果你不希望后续追加影响原数组，可以先复制。

```go
copyOfA := append([]int(nil), a...)
copyOfA = append(copyOfA, 100)
```

## 2. 删除：保持顺序的删除

删除下标 `i` 的元素，并保持后续元素顺序：

```go
s := []int{10, 20, 30, 40}
i := 1

s = append(s[:i], s[i+1:]...)
fmt.Println(s) // [10 30 40]
```

这行代码会把 `i+1` 后面的元素搬到前面。数据量大时，删除越靠前，搬移成本越高。

等价的 `copy` 写法更适合理解底层动作：

```go
s := []int{10, 20, 30, 40}
i := 1

copy(s[i:], s[i+1:])
s = s[:len(s)-1]

fmt.Println(s) // [10 30 40]
```

## 3. 删除：不保持顺序的快速删除

如果元素顺序不重要，可以用最后一个元素覆盖被删除位置。

```go
s := []int{10, 20, 30, 40}
i := 1

s[i] = s[len(s)-1]
s = s[:len(s)-1]

fmt.Println(s) // [10 40 30]
```

这个写法是 O(1)，适合任务集合、连接集合、待处理队列等不要求顺序的场景。

## 4. 插入：本质是扩容和搬移

在下标 `i` 插入一个元素：

```go
s := []int{10, 30, 40}
i := 1
v := 20

s = append(s, 0)      // 先扩出一个位置
copy(s[i+1:], s[i:])  // 把 i 之后的元素后移
s[i] = v

fmt.Println(s) // [10 20 30 40]
```

也可以写成一行，但可读性差一些：

```go
s = append(s[:i], append([]int{v}, s[i:]...)...)
```

这行会创建临时切片，初学者不建议优先使用。面试时说清楚“插入需要搬移元素”比背一行代码更重要。

## 5. 过滤：原地复用切片容量

过滤掉奇数：

```go
s := []int{1, 2, 3, 4, 5}
dst := s[:0]

for _, v := range s {
	if v%2 == 0 {
		dst = append(dst, v)
	}
}

fmt.Println(dst) // [2 4]
```

这会复用原底层数组，减少分配。适合过滤后不再需要原切片内容的场景。

如果原数据还要保留，就创建新切片：

```go
dst := make([]int, 0, len(s))
for _, v := range s {
	if v%2 == 0 {
		dst = append(dst, v)
	}
}
```

## 6. 删除指针元素后为什么可能要置零

切片缩短后，底层数组尾部可能还保存着旧引用。只要底层数组还活着，被引用对象就可能无法被 GC 回收。

```go
type Big struct {
	buf [1 << 20]byte
}

s := []*Big{&Big{}, &Big{}, &Big{}}

i := 1
copy(s[i:], s[i+1:])
s[len(s)-1] = nil // 清理尾部引用
s = s[:len(s)-1]
```

如果元素不是指针、map、slice、interface 等引用类数据，通常不需要专门清理。

## 7. 面试时怎么答

可以按操作分类回答：

- 追加用 `append`，但要说明它可能复用底层数组，也可能分配新数组。
- 保持顺序删除用 `append(s[:i], s[i+1:]...)` 或 `copy` 后缩短。
- 不保持顺序删除用最后一个元素覆盖，O(1)。
- 插入需要扩一个位置，再用 `copy` 后移元素。
- 过滤可以用 `dst := s[:0]` 原地复用，也可以新建切片保留原数据。
- 删除引用类元素后，长生命周期切片要考虑清理尾部引用。
