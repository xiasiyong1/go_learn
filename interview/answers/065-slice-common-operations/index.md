# 065. 切片操作

## 问题

Go 中切片的追加、删除、插入和过滤应该怎么写？

## 先给结论

切片没有内置删除或插入函数，常用写法都基于截取、append、copy 和复用底层数组。重点不是背模板，而是理解这些操作是否保持顺序、是否复用内存、是否清理被删除元素。

## 深入理解

### 1. 这道题真正考察什么

- 是否能写出保持顺序的删除和不保持顺序的快速删除。
- 是否知道过滤可以原地复用切片容量。
- 是否能解释删除指针元素后为什么可能要置零。
- 是否能判断 append 后原切片是否仍共享底层数组。

### 2. 底层机制要讲清楚

- 删除本质上是把不需要的元素从逻辑长度中移除。
- 保持顺序删除常用 `s = append(s[:i], s[i+1:]...)`。
- 不保持顺序删除可以用最后一个元素覆盖被删位置，再缩短长度。
- 如果底层数组仍保存指针元素，GC 会认为被引用对象仍可达。

### 3. 工程实践怎么取舍

- 需要保持顺序时使用 append 或 copy 删除。
- 不关心顺序且数据量大时，用尾元素覆盖减少搬移。
- 过滤大量元素时用 `dst := s[:0]` 原地构建结果。
- 删除包含指针或大对象引用的元素后，必要时把尾部元素置零。

### 4. 常见误区

- 遍历切片时正向删除元素，导致跳过后续元素。
- 删除后忘记清理尾部引用，造成内存滞留。
- 插入时没有预留容量，频繁扩容和复制。
- 把 append 写法用于多个共享切片，产生意外修改。

## 如何验证理解

- 写表格测试覆盖删除首、中、尾和空切片。
- 用 benchmark 比较保持顺序和不保持顺序删除的成本。
- 用 heap profile 验证删除指针元素后是否仍有对象被持有。
