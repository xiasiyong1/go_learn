# 006. Go GC 基础

## 问题

Go 的垃圾回收大致怎么工作？写代码时哪些行为会增加 GC 压力？

## 答案

Go 使用并发标记清扫垃圾回收。GC 会从根对象出发，标记仍然可达的对象，未被标记的对象会在之后被回收。

写代码时，GC 压力主要来自堆上对象数量和对象存活时间。分配越频繁、对象越多、存活越久，GC 需要扫描和管理的成本通常越高。

容易增加 GC 压力的行为：

- 在热点路径中频繁创建临时对象。
- 大量短生命周期对象逃逸到堆上。
- 长时间持有大对象引用。
- 不必要地把值装箱到接口中。
- 过度使用 `[]byte` 到 `string` 的来回复制。

## 优化方向

- 先用 benchmark 和 pprof 找到真实热点。
- 减少不必要分配，复用缓冲区。
- 控制对象生命周期，避免意外持有大对象。
- 理解逃逸分析结果，必要时调整数据结构或函数边界。
